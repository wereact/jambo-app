# CONTRIBUTING

## Writing your commit

When writing your description, follow these topics, listing changes made in each topic (_credits to angularjs team_):

- **build**: Changes that affect the build system or external dependencies (example scopes: gulp, broccoli, npm)
- **ci**: Changes to our CI configuration files and scripts (eg: `Jenkinsfile`)
- **docs**: Documentation only changes
- **feat**: A new feature
- **fix**: A bug fix
- **perf**: A code change that improves performance
- **refactor**: A code change that neither fixes a bug nor adds a feature
- **test**: Adding missing tests or correcting existing tests

> Note: If you made no changes related to some topic, just remove it.
